# ShipTools Data Table Specification

## Общая концепция
Основной элемент интерфейса `shiptools` — интерактивная таблица данных (Data Grid), отображающая характеристики 888 кораблей, загруженных с бэкенда.
Референсом выступает таблица с сайта `shiptool.st/params`.

В качестве основы будет использоваться **TanStack Table v8** (React Table) благодаря её производительности и гибкости (headless architecture). Стилизация — **Tailwind CSS 4**.

---

## 1. Структура колонок (Вкладка General)

На основе анализа HTML-кода `shiptools/spec/Ship parameters.html` (страница `?p=gen`), выделены следующие колонки:

| Имя колонки (Header) | Тип данных | Выравнивание | Сортировка | Описание / Пример |
| :--- | :--- | :--- | :--- | :--- |
| **Ship** | String | Left | А-Я, Я-А | Название корабля (Yamato, Montana). |
| **Tier** | Number | Center | 1-11 | Уровень корабля (10, 11). |
| **Class** | Enum/String| Center | По алфавиту | Класс (BB, CV, SS, CA, DD). |
| **Nation** | String | Center | По алфавиту | Нация (Japan, U.S.A., U.S.S.R.). |
| **Year** | Number | Right | По убыванию | Год проекта/спуска (1941). |
| **Length** | Number | Right | По убыванию | Длина в метрах (например, `263 m`). |
| **Beam** | Number | Right | По убыванию | Ширина в метрах (например, `38.9 m`). |
| **Tonnage** | Number | Right | По убыванию | Водоизмещение в тоннах (`72,800 t`). |
| **Detect. by sea**| Number | Right | По возрастанию | Заметность с кораблей (`17.46 km`). |
| **Detect. by air**| Number | Right | По возрастанию | Заметность с самолетов (`10.00 km`). |
| **Power / weight**| Number | Right | По убыванию | Удельная мощность (`2.06 hp/t`). |
| **Max speed** | Number | Right | По убыванию | Макс. скорость (`27.0 kn`). |
| **Acceleration** | Number | Right | По возрастанию | Время разгона (`39.1 s`). |
| **Rudder shift** | Number | Right | По возрастанию | Время перекладки руля (`22.1 s`). |
| **Turning radius**| Number | Right | По возрастанию | Радиус циркуляции (`900 m`). |

*Примечание:* Все числовые значения в UI должны форматироваться с соответствующими суффиксами (`m`, `t`, `km`, `hp/t`, `kn`, `s`), но сортировка должна происходить **по сырым числовым значениям** из базы данных.

## 2. Фильтрация и Управление

Таблица должна поддерживать следующие элементы управления над ней (или в сайдбаре):

### 2.1 Глобальные фильтры
- **Поиск по имени (Global Search):** Текстовое поле, фильтрующее колонку "Ship".
- **Фильтр по уровню (Tier):** Мультиселект (кнопки 1-11 или выпадающий список).
- **Фильтр по классу (Class):** Мультиселект (Линкор, Крейсер, Эсминец, Авик, Подлодка).
- **Фильтр по нации (Nation):** Мультиселект.

### 2.2 Сортировка
- Поддержка мульти-сортировки (Shift + Click по заголовку колонки).
- По умолчанию сортировка по `Tier` (убывание) -> `Class` -> `Nation` -> `Ship`.

---

## 3. Требования к UI/UX (Tailwind 4)

- **Липкая шапка (Sticky Header):** Заголовки колонок должны оставаться видимыми при прокрутке длинного списка из 888 кораблей.
- **Зебра (Zebra-striping):** Четные/нечетные строки должны иметь разный фон (`bg-slate-50` / `bg-white` в светлой теме) для удобства чтения.
- **Плотность (Density):** Компактный режим отображения (`py-1` или `py-2` для ячеек `<td>`), чтобы помещалось максимум данных на экран.
- **Hover эффекты:** При наведении на строку строка должна подсвечиваться (например, `hover:bg-blue-50`).
- **Трансформация данных:** Данные (суффиксы "km", "kn", разделители тысяч) добавляются на этапе рендеринга ячейки в TanStack Table (через `cell` аксессоры).

---

## 4. Архитектура работы с данными

Так как размер JSON-ответа с 888 кораблями невелик:
1. Фронтенд (React) делает **один** запрос `GET /api/ships` при загрузке.
2. Сохраняет весь массив в стейт (или Zustand).
3. TanStack Table работает в режиме `Client-Side` (локальная сортировка, фильтрация, пагинация отключена или выставлена на 100+ элементов). Это обеспечит мгновенную реакцию интерфейса на любые фильтры.
