# Редактор изображений (упрощенный)

### Ограничения:
1. Нет авторизации
2. Нет бэкэнда, ничего не сохраняется
3. Инструмент для разового редактирования, работает на клиенте в браузере (без нагрузки на сервер)

### Функционал:
1. Кнопка "Открыть изображение" с компьютера (открывается выбор файла), после выбора изображение загружается
2. Поле "загрузить url" текстовое, после enter или нажатия кнопки рядом (загрузить) изображение загружается
3. при нажатии ctrl (cmd) + V, если в буффере есть изображение -- оно загружается

Что такое "изображение загружается" -- поле редактирование очищается, изображение помещается в центр холста, если уже было загружено изображение -- все нарисованное удаляется,  изображение заменяется

### Что можно делать с изображением:
1. Выбор цвета (небольшая палитра ~10 основных цветов, по умолчанию красный), влияет на все интструменты рисования
2. Кисть с выбором размера, не блюра, ничего, просто тупая кисть, сглаживание нарисованного чуть-чуть -- использоваться будет для обведения
3. Прямоугольник - рисуется прямоугольник заданного цвета (только границы) -- это такой инструмент выделения
4. Маркер - Аналогично кисти, но полупрозрачный (тоже выделение классическое маркером), реализация multiply
5. Кроп - инструмент типа прямоугольник, после того как отпускаем мышку - изображение кропируется, есть галочка YOLO, по умолчанию включена и поведение как написано, если выключена -- сперва рисуется прямоугольник кропа, который можно перемещать углы и подтверждение enter. В обоих режимах должен работать ctrl+Z
6. ctrl (cmd) + A -- выделяет все, при нажатии backspace (del) удаляется все НАРИСОВАННОЕ (исходное изображение остается)
7. Зум колесиком/тачпадом


Должна быть поддержка истории рисования (ctrl (cmd) + Z -- отмена последнего нарисованного/кропа)

### Сохранение результата:
1. Кнопка скачать
2. ctrl (cmd) + A и ctrl (cmd) + C -- копирует изображение в буфер, просто ctrl+С без выделения тоже копирует все (с нарисованным)

### Интрефейс
Общий стиль с Notes -- надо использовать общие стили, панель инструментов вертикальная слева, большая часть окна -- холст справа

### Требвания к стеку
1. Использовать проверенные иструменты
2. Не забивать гвозди микроскопом -- не нужно тащить что-то очень тяжелое, приоритет простому и качественному (Konva.js ок)

### Основной сценарий

Быстрый редактор скриншотов, минимизация количества кликов, пользователь понимает что происходит, доп подтверждения "на всякий случай" не нужны
